# 修复导航栏被遮挡问题

## 1. 问题分析
- FluentWindow 内置了 NavigationView，包含 NavigationBar（侧边导航栏）和内容区域
- 自定义内容直接使用 `anchors.fill: parent` 覆盖了整个窗口，导致导航栏被遮挡
- 从截图可见窗口顶部有"Close Navigation"按钮，说明 NavigationView 已初始化但被覆盖

## 2. 修复方案

### 2.1 方案一：调整自定义内容布局
- 移除直接覆盖整个窗口的 Rectangle
- 利用 FluentWindow 内置的 navigationView 属性
- 确保自定义内容不覆盖 NavigationBar

### 2.2 方案二：调整 z-index 层级
- 确保 NavigationBar 的 z-index 足够高
- 调整自定义内容的 z-index，使其位于 NavigationBar 下方

### 2.3 方案三：重新设计布局结构
- 使用 FluentWindow 推荐的布局方式
- 利用内置的 NavigationView 和 StackView
- 避免直接覆盖整个窗口的元素

## 3. 实施步骤

### 3.1 修改 main.qml 文件
- 移除直接覆盖窗口的 Rectangle
- 简化界面结构
- 确保 NavigationBar 可见
- 调整内容区域的定位

### 3.2 调整 NavigationView 属性
- 确保 NavigationView 的 `appLayerEnabled` 为 true
- 调整 `navExpandWidth` 确保合适的导航栏宽度
- 确保 `z-index` 足够高

### 3.3 测试修复效果
- 运行应用程序
- 检查导航栏是否可见
- 测试导航功能是否正常
- 验证在不同屏幕尺寸下的表现

## 4. 预期效果
- 导航栏始终显示在左侧
- 自定义内容显示在导航栏右侧
- 导航功能正常可用
- 界面布局合理，无视觉冲突
- 适配不同屏幕尺寸

## 5. 文件修改
- 修改 `ui/main.qml` 文件
- 调整布局结构，确保导航栏可见
